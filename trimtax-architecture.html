Key Data Flows

⚡ §41.43 Comp Engine (estimate route)

1.Fetch subject property from properties by acct
2.Query comps: same neighborhood_code + similar sqft (±30%)
3.Fallback to zip code if <5 neighborhood comps
4.Compute $/sqft for each comp
5.Winsorize at 10th/90th percentile (remove outliers)
6.Median $/sqft × subject sqft = arguedValue
7.Confidence: comp count + CV + overage % → low/medium/high
8.Return savings = cadValue − arguedValue

📄 PDF Generation Flow (generate-pdf)

1.Receive submissionId + property data + base64 signature
2.Create new PDFDocument via pdf-lib
3.Page 1: Form 50-132 — Property Value Protest
4.  └ Fill: owner, address, county, CAD value, argued value
5.Page 2: Form 50-162 — Designated Agent Authorization
6.  └ Fill: same + embed signature image as PNG
7.Serialize PDF bytes → return as application/pdf download

🔍 Property Search Flow (search route)

1.Receive query param ?q= (partial address string)
2.SQL: WHERE site_addr_1 ILIKE '%q%' LIMIT 10
3.Uses idx_properties_addr B-tree index (fast scan)
4.Returns acct, address, city, zip, county, owner, cur_appr_val, bld_ar
5.Intake step 1 renders results as clickable autocomplete list

🔐 Auth Flow (Demo Mode)

1.Register: hash password → insert into users table
2.Login: lookup by email → verify hash → return token
3.Token format: btoa(email + ":" + timestamp)
4.Token stored in localStorage by client
5.Auth'd routes: check Authorization: Bearer <token> header
6.Decode token → extract email → look up userId
⚠Not production-safe — replace before launch
